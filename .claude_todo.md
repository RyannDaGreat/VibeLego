## In Progress — Unified Brick System Refactor

### Overview
Merge LEGO and Clara into ONE unified brick system. Clutch mechanism (tube/lattice/none)
becomes a dropdown, not a separate codebase. Add cross-shaped bricks (L/T/+) and
4-directional slopes. LEGO and Clara become presets of the unified system.

### Phase 1: Manifest + Architecture (before any code)
- [ ] Write complete requirements into manifest (all details from user conversation)
- [ ] Design unified file structure (what replaces lego_lib.py + clara_lib.py)
- [ ] Design unified panel_def.py SECTIONS layout
- [ ] Design new preset system (LEGO Standard, Clara Mini Brick, Clara Mini Slope, etc.)

### Phase 2: Unified Brick Builder (core geometry)
- [ ] Create `models/bricks/brick_lib.py` — single unified geometry library
- [ ] Port `lego_brick()` + `clara_brick()` → unified `brick()` with clutch param
- [ ] Port `lego_slope()` + `clara_slope()` → unified with 4-directional slopes
- [ ] Clutch dropdown: TUBE (LEGO tubes+ridges), LATTICE (Clara diagonal), NONE (hollow)
- [ ] Keep `common.py` as shared constants (unchanged)
- [ ] `bevel_above_z` already general — works as-is

### Phase 3: Cross-Shape Geometry
- [ ] Shape mode dropdown: RECTANGLE (current) vs CROSS
- [ ] CROSS mode params: studs_plus_x, studs_plus_y, studs_minus_x, studs_minus_y
- [ ] Cross arm width params: cross_width_x, cross_width_y (default 1, only visible in CROSS mode)
- [ ] All zeros = 1×1 at origin. Each delta extends outward.
- [ ] 2D footprint = union of two rectangles:
  - Horizontal bar: minus_x..plus_x extent, cross_width_y studs tall
  - Vertical bar: cross_width_x studs wide, minus_y..plus_y extent
  - Center overlap: cross_width_x × cross_width_y
- [ ] Example: plus_x=5, plus_y=5, rest=0, widths=1 → L-shape, 5+5+1=11 studs
- [ ] Shell: 2D polygon sketch (union of two rects) → extrude
- [ ] Cavity: inset polygon by WALL_THICKNESS → extrude to cavity_z → subtract
- [ ] Studs: placed at all grid positions within the cross footprint
- [ ] Tubes: placed at all (N-1)×(M-1) grid positions within the footprint
- [ ] Lattice: struts clipped to non-rectangular cavity via Mode.INTERSECT
- [ ] Text: on all stud tops within footprint
- [ ] RECTANGLE mode: backwards compatible, internally converts studs_x/studs_y to the
  directional system (plus_x = studs_x-1, plus_y = studs_y-1, minus = 0, etc. — or
  center the rectangle, details TBD)

### Phase 4: 4-Directional Slopes
- [ ] Replace single slope direction with 4: slope_plus_x, slope_plus_y, slope_minus_x, slope_minus_y
- [ ] Each has a flat_rows count (how many rows stay flat before slope begins)
- [ ] Multiple slopes active simultaneously (e.g., corner roof piece: minus_x + minus_y)
- [ ] Slope planes intersect — the brick is cut by ALL active slope planes
- [ ] Works on both RECTANGLE and CROSS shape modes
- [ ] New param: slope_min_z (default WALL_THICKNESS) — how low the slope descends
  - When 0, slope goes all the way to Z=0 (sharp edge at bottom)
  - Current hardcoded WALL_THICKNESS becomes the default value
- [ ] Cavity cut planes: each slope gets an offset plane (FLOOR_THICKNESS gap)
- [ ] Tubes/lattice clipped to the intersection of ALL sloped cavities

### Phase 5: Unified Panel + Parametric
